# 《AI 体态大师》技术方案草案

## 1. 产品定位与核心解决痛点

### 1.1 一句话定位

全球首款基于 **MCP 协议** 与 **端侧多模态大模型** 的软硬一体化“个人空间感知康复/运动智能体”。

### 1.2 核心解决痛点

* **瑜伽/普拉提运动代偿痛点**：普通人在无教练指导下练习极易发生“骨盆代偿借力”与“塌腰”。市面上的单目摄像头 AI 视觉极易受到**身体重叠、俯卧/仰卧视角遮挡**的限制，导致捕捉丢帧。
* **骨科术后离院康复黑盒**：脊柱侧弯矫正或关节置换术后患者，无法精准掌控关节活动度（ROM）的安全边界，动作过大易二次损伤，过小无效果，且缺乏定量数据供医生复查。
* **传统硬件穿戴校准繁琐与隐私泄露**：传统多点动捕需要极复杂的专业校准流程。若将用户穿紧身运动服的体态照片传到云端分析，存在极高的个人隐私泄露风险。

## 2. 硬件系统设计（零焊接乐高式架构）

系统采用“工字型弹力背带 + 魔术贴物理轨道”的模块化设计，适配不同身高体型的用户。

### 2.1 硬件节点拓扑

* **Node-C（主控/颈椎节点）**：放置于大椎穴（后脑勺下方凸起处）。内置 ESP32-C6 开发板、3.7V 锂电池与扁平振动马达 #1。
* **Node-T（胸椎节点）**：通过柔性硅胶线向后下方延伸，贴于胸椎正中，内置 BNO085 传感器。
* **Node-L（腰椎节点）**：贴于腰椎正中，内含 BNO085 传感器与微型振动马达 #2（用于腰椎代偿的即时体感拦截）。

### 2.2 核心硬件清单（总 BOM 成本约 210 元）

* **主控板**：ESP32-C6-DevKitC-1（支持 BLE 5.0 串行数据低功耗高速传输）。
* **传感器 ×3**：BNO085（内置 ARM Cortex-M0+ 处理器，直接硬件级输出游戏级精度的四元数，零消耗主控算力）。
* **多路复用器**：TCA9548A（解决三个 BNO085 芯片 I2C 硬件地址独占冲突，实现单总线轮询）。

## 3. 软件与算法架构（端云分层）

本系统严格遵守“高频、即时、隐私逻辑放端侧；低频、重度、生成式逻辑放云端”的现代端侧 AI 架构原则。

### 3.1 端侧（On-Device）核心引擎

* **视觉校准引擎（Qwen3-VL-2B-Instruct-MNN）**：
部署于手机本地的 1.3GB 压缩版多模态模型。用户侧身拍摄体态照，**完全离线**提取头前倾角度与圆肩形变度数，用作硬件传感器的绝对真值（Ground Truth）校准基线，照片用完即焚。
* **高频姿态融合算法（C++ JNI 层）**：
接收蓝牙推过来的 100Hz 原始四元数。使用相对旋转矩阵公式计算脊柱曲度：

$$Q_{local} = Q_{parent}^{-1} \otimes Q_{child}$$



将高维信号实时降维转换为可读的物理语义（如：`颈椎前倾 15°`）。
* **3D 实时可视化（Three.js WebView）**：
由全栈/前端思维驱动，在 App 内置 WebView 跑轻量级 Three.js，通过骨骼绑定（SkinnedMesh）低面数脊柱模型，实现 100Hz 高流畅度的动态跟练投影。

### 3.2 云端（Cloud）辅助支撑

* **Qwen-Max 大模型**：接收端侧本地 SQLite 沉淀的一周脱敏 angles 历史日志，进行深度医学推理，生成长篇个性化康复周报或高情商心理疏导对话。

## 6. 挑战赛突围与评委评审策略（Pitch Design）

### 6.1 针对 Arm 评委（性能与功耗指标，权重 25%）

* **硬核故事**：我们选择 BNO085 芯片在硬件层解决九轴融合运算，使 ESP32-C6 能够保持极高的 DeepSleep 唤醒效率；手机端利用 **MNN 推理引擎的 SME2 指令集优化**，大幅降低 Qwen3-VL 2B 在 Prefill 阶段的每帧功耗，证明软硬协同在边缘端的极高计算效能。

### 6.2 针对通义实验室评委（创意创新与前景，权重 40%）

* **硬核故事**：颠覆了传统“多模态大模型只能看图说话”的浅层应用，首次将大模型视作 **“物理传感器的绝对真值核验器（Ground Truth Calibrator）”**。通过 **MCP 标准管道**，让 Agent 真正具备了操作和理解物理空间运动学（Kinematics）的直觉。
