# 在 MNN 框架中开启 Arm SME2 加速 CPU 推理

**SME2（Scalable Matrix Extension v2）** 是 Armv9-A 架构推出的矩阵扩展指令集，针对矩阵乘累加（GEMM）做了大幅加速，是当前移动端 CPU 上运行 LLM、Diffusion 等大模型最高效的指令集之一。SME2 技术已获得领先厂商的广泛采用与认可。vivo、OPPO 等手机厂商已推出集成启用 SME2 的 Arm C1 CPU 与 Mali G1-Ultra GPU 的旗舰智能手机；此外，SME2 也已在 iOS 中大幅启用。

[MNN](https://github.com/alibaba/MNN) 作为阿里巴巴开源的轻量级端侧推理引擎，已完整支持 SME2 后端：在搭载 SME2 的手机上，Qwen2.5-1.5B 等 LLM 模型的 Prefill 性能相比传统 Armv8.6（Arm86，i8mm）指令集可提升 **2~3 倍**，显著降低首字延迟。

**开启方法非常简单，仅需两步：**

## 1. 编译期开启 Arm SME2 后端

MNN 默认即已开启 SME2 编译（`MNN_SME2=ON`），使用官方编译脚本（如 `project/android/build_64.sh`）构建出的 `libMNN.so` 即包含 SME2 后端。如需手动构建，确保 CMake 配置中带有：

```bash
cmake .. -DMNN_ARM82=ON -DMNN_SME2=ON -DMNN_BUILD_LLM=ON -DMNN_LOW_MEMORY=ON
```

## 2. 运行期无需任何额外配置

MNN 在初始化时会自动检测当前 CPU 是否支持 SME2 指令，若支持则自动调度到 SME2 内核执行 GEMM/GEMV 算子；不支持的设备则自动回退到 Armv8.6（i8mm）/ Armv8.2（dotprod）等内核，保证同一份代码可跨设备兼容。

因此，开发者按 MNN 常规方式加载模型并推理即可，无需修改业务代码：

```cpp
// LLM 推理示例
auto llm = MNN::Transformer::Llm::createLLM(config_path);
llm->load();
llm->response("你好");
```

如需快速验证 Arm SME2 是否生效，可使用 MNN 自带的 LLM Benchmark 工具：

```bash
./llm_bench -m /path/to/MODEL/config.json
```

观察 Prefill 阶段的 token/s 即可直观对比 Arm SME2 带来的性能提升。

更多构建选项与 LLM 使用细节，详见 MNN 官方文档：<https://github.com/alibaba/MNN>。
