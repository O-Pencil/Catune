# 硬件单板自检指南（ESP32-S3 / ESP32-C6）

> 版本：2026-06-28  
> 用途：杜邦线 / BNO085 到货前或到货后，**先单测开发板**能否烧录、串口、LED；再接 IMU 与正式固件。  
> 关联：[BLE协议与固件](./BLE协议与固件.md) · [硬件采购与小白使用指南](./硬件采购与小白使用指南.md)

---

## 1. 你手上的两块板

| 板子 | 项目主线路径 | 说明 |
| --- | --- | --- |
| **ESP32-S3** | ✅ **主推** | 仓库 `catune_node.ino` 默认按 S3 编写（I2C **GPIO8/9**，BLE 正常） |
| **ESP32-C6** | 🟡 可烧录自检 | 参赛演示物料里有 C6；**正式姿态固件需改 I2C 引脚 + 板型**（见 §5） |

PRD 已从 C6 切到 S3（双核 + PSRAM 更稳）。**先分别单测两块板能亮、能串口**，再优先在 **S3** 上接 BNO085 跑 `catune_node`。

---

## 2. 工具准备（Mac）

### 2.1 Arduino IDE（推荐新手）

1. 安装 [Arduino IDE 2.x](https://www.arduino.cc/en/software)
2. **设置 → 附加开发板管理器网址** 添加（若还没有）：
   ```
   https://espressif.github.io/arduino-esp32/package_esp32_index.json
   ```
3. **工具 → 开发板 → 开发板管理器** → 安装 **esp32 by Espressif Systems**（建议 **≥ 3.0**）
4. USB-C **数据线**接 Mac（必须是数据线，不能是纯充电线）

### 2.2 命令行（可选）

```bash
brew install arduino-cli
arduino-cli core update-index
arduino-cli core install esp32:esp32
```

## 3. 阶段 A：单板测试（不接 BNO085）

固件：`firmware/board_test/board_test.ino`  
作用：串口打印板型、闪 LED，确认 **USB + 烧录 + 串口** 正常。

### 3.1 ESP32-S3

| 项 | 设置 |
| --- | --- |
| 开发板 | **ESP32S3 Dev Module** |
| USB CDC On Boot | **Enabled**（串口走 USB 时） |
| 端口 | 选出现的 `/dev/cu.usbmodem*` 或 `usbserial*` |

步骤：

1. 打开 `firmware/board_test/board_test.ino`
2. 开发板选 **ESP32S3 Dev Module** → **上传**
3. **工具 → 串口监视器**，波特率 **115200**

**通过标准：**

```
=== Catune 单板自检 ===
芯片: ESP32-S3 …
每 1s 打印 tick …，板载 LED 闪烁
```

### 3.2 ESP32-C6

| 项 | 设置 |
| --- | --- |
| 开发板 | **ESP32C6 Dev Module** |
| 端口 | 同上，换 C6 板时重新选端口 |

步骤同 S3，仅改开发板型号。

**通过标准：** 串口有 `ESP32-C6` 字样 + tick 输出；LED 闪（部分 C6 板 LED 引脚不同，**以串口为准**）。

### 3.3 常见问题

| 现象 | 处理 |
| --- | --- |
| 找不到端口 | 换 USB 口/换数据线；S3 可能需要按 **BOOT** 再点上传 |
| 上传失败 `Failed to connect` | 按住 **BOOT** → 点上传 → 出现 Connecting 时松开 |
| 串口乱码 | 波特率改为 **115200** |
| 串口无输出 | S3 打开 **USB CDC On Boot: Enabled**；重启板子 |

## 4. 阶段 B：I2C 扫描（已接 BNO085 时）

仍可在 `board_test` 之后，烧录正式固件做 I2C 自检：

- S3：打开 `firmware/catune_node/catune_node.ino`，板型 **ESP32S3 Dev Module**
- 接线：`VIN→3V3`、`GND→GND`、`SDA→GPIO8`、`SCL→GPIO9`

上电后串口应出现：

```
I2C 扫描…
  发现设备 @ 0x4A  <- BNO085
BNO085 OK
Catune-Node 广播中…
```

若无 `0x4A`：查 SDA/SCL 是否接反、是否共 GND、是否用了 5V。

---

## 5. ESP32-C6 跑正式固件（可选，非默认）

`catune_node.ino` 当前写死 S3 引脚。C6 要用需改：

```cpp
// ESP32-C6-DevKitC-1 常用（以你板子丝印为准，不对就查 Espressif 引脚图）
#define I2C_SDA 6
#define I2C_SCL 7
```

Arduino：**开发板 → ESP32C6 Dev Module**，再上传 `catune_node.ino`。

> C6 与 S3 BLE 栈在 Arduino 下均可用；App 只认 Service UUID，与主控型号无关。

## 6. 阶段 C：与 App 联调

1. 手机安装 Catune（Android 可先测；iOS 待 [iOS真机构建与环境](./iOS真机构建与环境.md) Xcode 26 就绪）

4. 坐直 → **坐直校准**；Monitor 看颈/胸/腰数据

## 7. 建议顺序（今天）

```
1. S3 → board_test 烧录 → 串口 OK
2. C6 → board_test 烧录 → 串口 OK
3. （有 BNO085）S3 → catune_node + I2C 扫描 0x4A
4. App BLE 连接
```

两块板都通过 **阶段 A** 后，至少说明：**USB、烧录器、芯片本身可用**。
