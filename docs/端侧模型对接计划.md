## 1. 为什么当前代码暂不集成端侧模型

初赛第一优先级是「RN 框架能在模拟器跑起来 + 有基础页面显示关键数据」。端侧 Qwen + MNN 推理在当前开发机上**无法落地**，原因有二：

1. **模型装不下**：Qwen 2B INT4 权重（约 1.2–1.5GB）+ MNN 运行库不适合放进当前开发/CI 环境，也无法推到模拟器跑通。
2. **原生构建缺源**：原 `CMakeLists.txt` 把 `MNN_SOURCE_ROOT` 硬编码到仓库外的 `Template-github/eyes-on-my-phone/MNN-master`，该路径在本仓库不存在，导致 `externalNativeBuild` 必然失败，`assembleDebug` 连 APK 都产不出来。

因此初赛阶段**先把端侧推理从主链路移除**，让 App 成为「纯 RN + 轻量 Kotlin」可构建、可在 x86_64 模拟器运行的形态，由本计划规划重新接入。

## 2. 本次移除了什么（均可从 git 恢复）

| 移除项 | 原路径 | 恢复方式 |
| --- | --- | --- |
| C++/JNI 推理桥 | `android/app/src/main/cpp/`（`eyes_mnn_bridge.cpp` / `eyes_llm_session.cpp` 等） | `git checkout d17af44 -- android/app/src/main/cpp` |

| CMake / NDK 构建配置 | `app/build.gradle` 的 `externalNativeBuild`、`abiFilters "arm64-v8a"` | 见本计划 §4 |

> 注：`MainActivity.calculateSpineAnglesStatic` 已保留为**纯 Kotlin 占位算法**，作为四元数→姿态角的回退实现；`KinematicsHub` 状态机完全保留。

## 3. 目标架构（重新接入后）

```text
RN UI (App.tsx)
  └─ KinematicsModule (RN 桥) ── 订阅 ── KinematicsHub(StateFlow, 规则状态机)
                                              ▲
                  BLE/模拟四元数 ──► 角度解算 ─┘
                                              │ 结构化 Prompt（角度序列 + 状态）
                                              ▼
                                   InferenceExecutor（单线程 eyes-mnn-infer）
                                              │ JNI
                                              ▼
                                   libposture_ai_bridge.so → libMNN.so
                                              │
                                   Qwen 2B INT4 本地 CPU 推理（SME2/NEON）
                                              │ 仅结构化字段
                                              ▼
                          本地查表生成文案 → 禁词检查 → RN 展示
```

**纪律（沿用 PRD §5.10）**：姿态分类/提醒走规则（KinematicsHub），模型只做「文案生成」这类规则做不到的事；模型超时/崩溃/OOM 自动回退规则引擎；模型输出必须过安全链。

## 4. 分步接入计划

### 阶段 0 · 环境与素材（前置，需在有算力/存储的机器上）
- [ ] 准备支持 **Arm SME2** 的 MNN arm64 运行库（`libMNN.so`），并保留 NEON 回退。
- [ ] 获取 / 转换 Qwen 2B INT4 姿态分类模型为 MNN 格式（`config.json` / `llm.mnn` / `llm.mnn.weight`）。
- [ ] 准备 MNN 源码（含 `llm/llm.hpp` 等头文件），供 JNI 编译。

### 阶段 1 · 恢复原生构建
- [ ] 从 git 恢复 `cpp/` 与 `jniLibs/`（见 §2）。
- [ ] 修正 `cpp/CMakeLists.txt` 的 `MNN_SOURCE_ROOT` 为**仓库内相对路径**，不再指向外部目录。
- [ ] 在 `app/build.gradle` 重新加入：
  ```gradle
  externalNativeBuild { cmake { cppFlags "-std=c++17"; arguments "-DANDROID_STL=c++_shared" } }
  // arm64-v8a 跑真机端侧推理；x86_64 模拟器无 libMNN，需 loadLibrary 失败时回退规则
  ```
- [ ] `abiFilters` 视情况：端侧推理只在 `arm64-v8a` 真机演示；x86_64 模拟器保持「无模型 → 回退占位算法」。

### 阶段 2 · 推理管线
- [ ] 恢复 `InferenceExecutor` + `MnnPerceptionEngine`，固化输入/输出契约（见 [技术实现文档 §3.3](./技术实现文档.md)）：
  - 输入：`{ window_ms, neck_pitch_deg, thor_pitch_deg, lumbar_roll_deg, duration_min, last_state }`
  - 输出：`{ classification, confidence, action_id, severity_level }`
- [ ] `MainActivity.calculateSpineAnglesStatic` 接回真实四元数→姿态角算法（替换占位实现）。
- [ ] 模型缺失/超时/OOM → 回退到 `KinematicsHub` 规则结果，主链路不中断。

### 阶段 3 · 模型安装与运行
- [ ] 推送模型到设备：
  ```bash
  adb push <本地模型目录>/ /data/data/com.catune/files/mnn_models/qwen-posture-2b-int4/
  ```
  （目录名建议从历史名 `qwen3-vl-2b` 统一为 `qwen-posture-2b-int4`）

### 阶段 4 · 指标与安全链
- [ ] App 状态页 / 调试面板暴露：MNN 库已加载、模型已加载、`backend = SME2|NEON|unknown`、`ttft_ms` / `decode_tps` / `tokens_generated`。
- [ ] 端侧路径安全链：结构化字段 → 本地查表 → 禁词检查 → 命中替换为预置兜底。

## 5. 验收标准（对齐 PRD K6 与技术文档 §9）

- [ ] 关网状态下触发一次姿态异常，App 展示本地分类 + 文案 + MNN 推理指标。
- [ ] 首 token ≤ 1.5s，完整文案 ≤ 3s（骁龙 7+ 级别）。
- [ ] 模型缺失时展示 `degraded_mode`，主流程不崩溃。
- [ ] SME2 设备使用支持 SME2 的 MNN 库；非 SME2 设备回退 NEON 可启动。
- [ ] x86_64 模拟器无模型时，UI 仍可跑通规则状态机与 F7 模拟数据。
