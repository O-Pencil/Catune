# Patterns

## Settings Diagnostic Card

Use for advanced state like MNN, BLE, model download, or assessment backend.

- Keep status visible.
- Keep primary command clear.
- Put raw/debug detail lower in the card.
- Never let diagnostics dominate Desk.

## Focused Overlay

Use for Training, Assess, and Onboarding.

- One job per overlay.
- Clear close/finish path.
- Preserve the user's return context.
- Avoid adding unrelated navigation.

## Data Source Choice

Use segmented/chip-like controls or clearly grouped buttons for sensor/mock/BLE choices.

- Show active source.
- Show unavailable/error state without crashing.
- Keep mock available for demo/dev.

## Advice With Action

Advice should pair a short sentence with a concrete next action when possible.

- Good: "收下巴，肩膀轻轻向后喵～"
- Weak: "请注意姿势"
- Bad: "治疗你的颈椎问题"
